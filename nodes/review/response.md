**Findings**

- Critical: runtime Hermes auth still bypasses the authority-token boundary. [`hermes.py`](/home/r/.fabro/runs/20260322-01KMC096ZYSY842CACE23WGX45/worktree/services/home-miner-daemon/hermes.py#L181) authorizes by `hermes_id` only, and the live endpoints in [`daemon.py`](/home/r/.fabro/runs/20260322-01KMC096ZYSY842CACE23WGX45/worktree/services/home-miner-daemon/daemon.py#L236) and [`daemon.py`](/home/r/.fabro/runs/20260322-01KMC096ZYSY842CACE23WGX45/worktree/services/home-miner-daemon/daemon.py#L319) use that path. In practice, knowing a paired ID is enough to read status, append summaries, and read events without presenting the token from `/hermes/connect`.
- High: token expiry is not trustworthy. [`hermes.py`](/home/r/.fabro/runs/20260322-01KMC096ZYSY842CACE23WGX45/worktree/services/home-miner-daemon/hermes.py#L108) stores `token_expires_at` as “now”, and [`hermes.py`](/home/r/.fabro/runs/20260322-01KMC096ZYSY842CACE23WGX45/worktree/services/home-miner-daemon/hermes.py#L147) never enforces expiry anyway.
- Medium: the planned boundary tests are still missing. `python3 -m pytest services/home-miner-daemon/tests/test_hermes.py -v` failed because that file does not exist.
