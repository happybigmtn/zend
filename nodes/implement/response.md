`★ Insight ─────────────────────────────────────`
The test fixture chain is critical: `daemon_url` depends on `daemon_process` which actually starts the daemon. Without this dependency, pytest would not invoke the daemon startup code because fixture execution follows dependency chains — if no test explicitly requests a fixture, it never runs. This is a common pytest pitfall when using session/module-scoped fixtures for services.
