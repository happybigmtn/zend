### Key Design Notes

- **Canonical port 8080** — hardcoded to ensure deterministic behavior regardless of inherited environment
- **Simulator pattern** — daemon exposes same contract a real miner backend will use
- **Observe-only default capability** — paired devices can observe but not control
- **Deferred to slice 2:** `/miner/set_mode`, LAN binding, auth adapter integration, Hermes integration

`★ Insight ─────────────────────────────────────`
The `${VAR:-default}` bash pattern caused the port mismatch. When a parent process exports `ZEND_BIND_PORT=18080`, child scripts inherit it even if they set a default. Hardcoding the value inside the script ensures determinism independent of environment inheritance — a common source of production bugs.
